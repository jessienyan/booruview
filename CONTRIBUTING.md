# Contributing

Contributions are welcome!

They don't have to be code changes. If you have ideas or feedback I want to hear about them.

## Code Contributions

*BEFORE* starting any work, please make sure you've done the following:

1. You've claimed an issue and said you will work on it
2. You've created a new issue if there wasn't an existing one already

This prevents duplicate/overlapping work as well as potentially undesired changes.

As for the actual code:

1. It should match the existing style and formatting
2. Tests are added if they are applicable (not required but nice to have)
3. NO dependencies should be added unless they are discussed beforehand
4. You must be able to explain the changes in your own words
